CRAWLING PROBLEM
쇼핑몰마다 다른 딥링크와 접근 정책

OneLink, Airbridge, App Link가 실제 상품 페이지가 아닌 중간 페이지로 연결됐고, 올리브영 모바일 페이지에서는 차단 화면이 반환됐습니다.

CRAWLING SOLUTION
UrlResolver와 실패 격리

링크 유형 식별, 리디렉션 추적, PC 상세 페이지 정규화를 별도 계층으로 분리했습니다. 실패 시 서버 500 대신 title과 price를 null로 반환했습니다.

CONCURRENCY PROBLEM
동일 만료 대상의 중복 알림 생성

여러 실행 흐름이 notified=false 상태를 동시에 조회한 뒤 각각 알림을 생성하는 경쟁 상태가 발생했습니다.

CONCURRENCY SOLUTION
DB 조건부 갱신을 통한 원자적 선점

claimNotified()가 상태 변경에 성공한 실행만 알림을 만들도록 구성하고, clearAutomatically와 flushAutomatically로 영속성 컨텍스트를 동기화했습니다.
